About

This web application was built using Django and showcases several core backend features, along with a clean and intuitive user interface. It is intended as a demonstration of my practical skills in building full-stack web apps with Python and Django.

Key Features & How to Use:

  • User Management

    Visitors can register a new account or log in with existing credentials. After logging in, users gain access to protected parts of the site such as the weather page or personal thoughts.

  • Posts (My Thoughts)

    Authenticated users can create and view personal posts â€“ these can be simple reflections, notes, or diary entries. Each post includes a title, content body, optional banner image, and is automatically timestamped.

  • Weather Page

    Users can search for current weather conditions automatically for their city or different by entering a city name. The app fetches real-time data and displays basic weather information.

  • Navigation & Layout

    The app features a consistent navigation bar with dropdown menus for grouped links (e.g., under "Users" or "Posts"). Layout and styling are handled with custom CSS for clarity and usability.

Technical Overview

  • Django class-based and function-based views
  • Dynamic forms with model integration
  • Custom user model support
  • URL routing, templates, and static file handling
  • CSRF protection, form validation, user sessions

Weather Setup

To enable weather data functionality, you will need an API key from OpenWeather. Follow these steps:

  1. Create an account at openweathermap.org.
  2. Navigate to the API section and generate your free API key.
  3. Create a .env file in your Django project root (if it doesn't exist).
  4. Add the following line to your .env file:
    WEATHER_API_KEY=your_api_key_here
  5. Make sure your Django app loads environment variables securely.
